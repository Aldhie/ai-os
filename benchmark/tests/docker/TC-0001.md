## Question

```yaml
# Bug report: app container cannot reach database. 
# Health checks are passing but app logs show:
# "Connection refused: postgres:5432"

version: '3.8'
services:
  app:
    image: myapp:latest
    ports:
      - "8080:8080"
    environment:
      - DATABASE_URL=postgresql://user:pass@postgres:5432/mydb
    depends_on:
      - postgres
    networks:
      - frontend
  
  postgres:
    image: postgres:15
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
      - POSTGRES_DB=mydb
    volumes:
      - pgdata:/var/lib/postgresql/data
    networks:
      - backend
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "user"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  pgdata:

networks:
  frontend:
  backend:
```

Find ALL bugs. Provide corrected docker-compose.yml and explain each fix.
```

---

## Expected Behaviour

1. **Root cause**: `app` is on `frontend` network; `postgres` is on `backend` network. They are on **different isolated networks** — they cannot reach each other. DNS name `postgres` is only resolvable within the same network.
2. **Fix**: Either put both on the same network, or add `app` to `backend` network as well.
3. **Secondary issue**: `depends_on` without `condition: service_healthy` only waits for container start, NOT for postgres to be ready to accept connections. The health check exists but is not being used as a dependency condition.
4. **Fix 2**: Add `depends_on: postgres: condition: service_healthy`
5. **Optional observation**: No resource limits (memory, CPU) — postgres could OOM under load.

**Corrected relevant section:**
```yaml
  app:
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - frontend
      - backend  # Added
  
  postgres:
    networks:
      - backend
```
