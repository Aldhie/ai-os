## Question

```
Design a production RAG system for an enterprise knowledge base:
- 5 million documents, average 2 pages each
- Documents updated daily (10% churn)
- 1,000 queries/hour
- Answer accuracy target: >85% on internal evals
- Latency: <3 seconds end-to-end
- Languages: English and Indonesian

Deliver:
1. Full pipeline architecture (ingestion + retrieval + generation)
2. Embedding model selection with justification
3. Vector database selection with justification
4. Chunking strategy
5. How to handle the 10% daily document churn
6. How to measure and improve answer accuracy over time
```

---

## Expected Behaviour

1. **Pipeline**: Document loader → chunker → embedding model → vector store + metadata index → hybrid retrieval (dense + sparse) → reranker → LLM generation
2. **Embedding**: multilingual model (e.g., `nvidia/nv-embedqa-e5-v5` or `multilingual-e5-large`) — must address English + Indonesian requirement
3. **Vector DB**: Qdrant, Weaviate, pgvector, or Milvus — justified by scale (10M+ vectors after chunking)
4. **Chunking**: Semantic or recursive chunking at ~512 tokens with overlap; NOT fixed-size without overlap
5. **Daily churn**: Incremental indexing with document versioning and deletion; NOT full re-index daily
6. **Accuracy measurement**: Evaluation dataset with ground-truth Q&A pairs; RAGAS or similar framework; human review sample
