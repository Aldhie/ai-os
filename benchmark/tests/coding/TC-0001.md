## Question

```
Implement a thread-safe LRU (Least Recently Used) cache in Python with:
- O(1) get and put operations
- Maximum capacity (evict LRU entry when full)
- Per-entry TTL (time-to-live) in seconds
- Thread-safe for concurrent reads and writes
- get() returns None for expired or missing keys
- Support for a stats() method returning: hits, misses, evictions, expirations

Requirements:
- No external libraries except threading and time from stdlib
- Must include at least 5 unit tests demonstrating correctness
- Include a brief explanation of your concurrency approach
```

---

## Expected Behaviour

**Core implementation:**
```python
from collections import OrderedDict
import threading
import time

class LRUCache:
    def __init__(self, capacity: int, default_ttl: float = None):
        self.capacity = capacity
        self.default_ttl = default_ttl
        self.cache: OrderedDict = OrderedDict()  # key -> (value, expire_at)
        self._lock = threading.RLock()
        self._stats = {'hits': 0, 'misses': 0, 'evictions': 0, 'expirations': 0}
    
    def get(self, key):
        with self._lock:
            if key not in self.cache:
                self._stats['misses'] += 1
                return None
            value, expire_at = self.cache[key]
            if expire_at and time.monotonic() > expire_at:
                del self.cache[key]
                self._stats['expirations'] += 1
                self._stats['misses'] += 1
                return None
            self.cache.move_to_end(key)
            self._stats['hits'] += 1
            return value

    def put(self, key, value, ttl: float = None):
        with self._lock:
            effective_ttl = ttl if ttl is not None else self.default_ttl
            expire_at = time.monotonic() + effective_ttl if effective_ttl else None
            if key in self.cache:
                self.cache.move_to_end(key)
            self.cache[key] = (value, expire_at)
            if len(self.cache) > self.capacity:
                self.cache.popitem(last=False)
                self._stats['evictions'] += 1

    def stats(self):
        with self._lock:
            return dict(self._stats)
```

Key correctness points:
- `OrderedDict` for O(1) LRU ordering
- `RLock` (not Lock) to allow reentrant calls
- `monotonic()` not `time()` for TTL (immune to clock adjustments)
- Eviction happens AFTER insertion (correct LRU behaviour)
- TTL check on every get (lazy expiry)

## Failure Conditions

- Uses a plain `dict` (no ordering guarantee pre-Python 3.7 — acceptable to note this)
- No locking at all
- put() checks capacity before insertion (evicts wrong entry)
- TTL uses `time.time()` without noting the clock skew risk
