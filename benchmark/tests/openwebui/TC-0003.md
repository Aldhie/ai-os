## Question

```
Open WebUI does not natively support the `extra_body` parameter required
to enable thinking mode in NVIDIA Nemotron Ultra via the NIM API.

Design and implement a complete Open WebUI Pipeline (Python) that:
1. Intercepts outgoing requests before they reach NIM
2. Detects whether the user's message starts with /think or /nothink
3. Injects the appropriate extra_body into the request:
   - /think → extra_body: {"thinking": {"type": "enabled", "budget_tokens": 10000}}
   - /nothink → extra_body: {"thinking": {"type": "disabled"}}
   - default (no prefix) → no extra_body injection
4. Strips the /think or /nothink prefix from the actual message
5. Passes the modified request to the NIM API endpoint

Provide the complete Pipeline class with proper Open WebUI Pipelines API structure.
```

---

## Expected Behaviour

```python
from typing import List, Optional
from pydantic import BaseModel
import json

class Pipeline:
    class Valves(BaseModel):
        NIM_API_KEY: str = ""
        NIM_BASE_URL: str = "https://integrate.api.nvidia.com/v1"
        DEFAULT_BUDGET_TOKENS: int = 10000
    
    def __init__(self):
        self.name = "NIM Thinking Mode Pipeline"
        self.valves = self.Valves()
    
    async def on_startup(self):
        pass
    
    async def inlet(self, body: dict, user: Optional[dict] = None) -> dict:
        messages = body.get("messages", [])
        if not messages:
            return body
        
        last_message = messages[-1]
        content = last_message.get("content", "")
        
        if content.startswith("/think"):
            body["extra_body"] = {
                "thinking": {
                    "type": "enabled",
                    "budget_tokens": self.valves.DEFAULT_BUDGET_TOKENS
                }
            }
            last_message["content"] = content[len("/think"):].strip()
        elif content.startswith("/nothink"):
            body["extra_body"] = {"thinking": {"type": "disabled"}}
            last_message["content"] = content[len("/nothink"):].strip()
        
        return body
```

## Failure Conditions

- Uses a non-standard Pipeline structure
- `extra_body` injected at wrong level (e.g., inside `messages`)
- Prefix not stripped from user message
- Missing Valves class for API key configuration
