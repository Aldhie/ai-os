## Question

```
Simulate a two-session scenario.

Session 1 (yesterday):
User: "I prefer responses in Indonesian. I'm a software engineer with 5 years 
experience. I like concise answers with code examples. Don't give me basic 
explanations. I use Python primarily."

Session 2 (today, new conversation):
User: "Explain how async/await works."

Expected behavior: The AI should recall Session 1 preferences and respond:
- In Indonesian
- At senior engineer level (no basic explanations)
- Concisely with a Python code example
- Without asking the user to re-state their preferences

Evaluate: Does the AI demonstrate correct memory-informed behavior?
```

---

## Expected Behaviour

1. Response is in Indonesian (primary preference)
2. No "async/await is a way to write asynchronous code" preamble (senior level)
3. Includes Python `async def` / `await` code example
4. Does not ask "What language do you prefer?" or "What is your experience level?"
5. Response is concise (≤500 words)

---

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Responds in Indonesian | 30% | Explicit preference honored |
| Senior-level framing | 25% | No basic definitions |
| Python code example included | 25% | Explicit preference honored |
| No re-prompting for preferences | 20% | Memory working correctly |

---

## Failure Conditions

- Responds in English
- Starts with basic definition of async
- Asks user to re-state their preferences
- No code example
