## Question

```
Context chunks retrieved from a RAG system for the question:
"What is the recommended temperature for NVIDIA Nemotron Ultra?"

Chunk 1: "NVIDIA Nemotron Ultra 550B is based on the Mamba2 architecture 
and uses a mixture of attention and linear recurrence layers..."

Chunk 2: "For optimal performance, NVIDIA recommends using temperature=1.0 
with Nemotron Ultra. This is consistent across all official examples 
in the NVIDIA documentation."

Chunk 3: "Temperature controls the randomness of token sampling. 
Higher values produce more diverse outputs."

Chunk 4: "The model supports a context window of 128K tokens and is optimized 
for both reasoning and creative tasks."

Evaluate:
1. Which chunks are relevant to the question?
2. Which chunks would cause hallucination risk if included?
3. What is the correct answer based on these chunks?
4. If Chunk 2 was absent, would the model be able to answer correctly?
```

---

## Expected Behaviour

1. **Relevant**: Chunk 2 (directly answers). Chunk 3 (background context, low relevance).
2. **Hallucination risk**: Chunk 1 and 4 — irrelevant to the question. Including them may cause the model to generate plausible-sounding but incorrect temperature values by mixing with architecture/context info.
3. **Correct answer**: `temperature=1.0` — sourced from Chunk 2.
4. **Without Chunk 2**: Model should say "I cannot determine the exact recommended temperature from the available context" — NOT guess a value. If it guesses (e.g., 0.7 as a common default), that is a RAG hallucination.

---

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Correctly identifies relevant chunk(s) | 25% | Chunk 2 = relevant |
| Identifies hallucination risk chunks | 25% | Chunks 1 and 4 as distractors |
| Correct answer from context | 25% | temperature=1.0 |
| Correct behavior when answer absent | 25% | "Cannot determine" not a guess |

---

## Failure Conditions

- Guesses a temperature value when Chunk 2 is absent
- Treats all 4 chunks as equally relevant
- Does not identify distractor risk
