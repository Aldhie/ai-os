## Question

```
You are a senior engineering manager. You have been asked to deliver a 
production-ready REST API platform in 90 days with:

- Team: 4 backend engineers, 1 DevOps engineer, 1 QA engineer
- Requirements:
  * User authentication (OAuth2 + JWT)
  * 10 resource endpoints (CRUD)
  * Rate limiting (per user, per IP)
  * Audit logging
  * Horizontal scaling capability (target: 1000 RPS)
  * CI/CD pipeline
  * 99.5% uptime SLA
  * Full API documentation
  * Staging + Production environments

Deliver:
1. A Work Breakdown Structure (WBS) with phases
2. The critical path through the project
3. The top 5 risks with mitigations
4. Which requirements are non-negotiable vs. negotiable given the 90-day constraint
```

---

## Expected Behaviour

1. Model produces a coherent 3-4 phase WBS: Foundation → Core Features → Hardening → Launch
2. Model identifies critical path: infrastructure setup → auth → core endpoints → load testing → production deploy
3. Model identifies real risks: scope creep, auth complexity underestimation, performance at scale, key person dependency, infra delays
4. Model correctly classifies non-negotiable (auth, core endpoints, CI/CD, staging) vs negotiable (audit logging depth, documentation completeness, rate limiting sophistication)

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| WBS quality | 30% | Phases are logical, complete, properly ordered |
| Critical path accuracy | 25% | Identifies the actual bottleneck sequence |
| Risk quality | 25% | Specific, actionable risks with mitigations |
| Negotiability analysis | 20% | Reasonable classification with justification |
