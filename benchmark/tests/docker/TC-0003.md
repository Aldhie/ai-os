## Question

```
Design a production Docker Compose stack that:
- Runs Open WebUI (latest)
- Connects to NVIDIA NIM inference microservice
- Supports GPU passthrough for NIM container
- Includes Ollama as fallback for offline use
- Persistent volumes for: Open WebUI data, model weights
- Health checks on all services
- Environment variable management (no hardcoded secrets)
- Restart policies for production

Provide the complete docker-compose.yml with comments explaining each decision.
```

---

## Expected Behaviour

Key expected elements:
- `deploy.resources.reservations.devices` with `capabilities: [gpu]` for NIM service
- `NVIDIA_VISIBLE_DEVICES=all` environment variable
- Open WebUI `OPENAI_API_BASE_URL` pointing to NIM endpoint
- `WEBUI_SECRET_KEY` from environment variable, not hardcoded
- Named volumes for `open-webui` and NIM model cache
- `healthcheck` on all 3 services
- `restart: unless-stopped` (not `always` — prevents boot loop on misconfigured containers)
- `.env` file pattern for secrets, not inline values
