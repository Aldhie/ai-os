## Question

```
Implement an async HTTP client wrapper in Python (using aiohttp or httpx) that:
- Retries failed requests with exponential backoff + jitter
- Implements a circuit breaker (open after 5 failures in 60 seconds)
- Circuit breaker half-open state: allows 1 test request after 30s
- Returns structured errors (not raw exceptions)
- Logs each retry with reason and attempt number
- Type-annotated

Include a usage example showing:
- Normal successful call
- Retry behaviour on 503
- Circuit breaker opening after repeated failures
```

---

## Expected Behaviour

- `CircuitBreaker` class with states: CLOSED, OPEN, HALF_OPEN
- Exponential backoff: `min(base * (2 ** attempt) + random.uniform(0, 1), max_wait)`
- Circuit opens after `failure_threshold` failures within `window_seconds`
- Circuit transitions OPEN → HALF_OPEN after `recovery_timeout` seconds
- HALF_OPEN allows exactly 1 probe request; success → CLOSED, failure → OPEN
- `AsyncAPIClient` wraps the circuit breaker and handles retry logic
- `APIError` dataclass with: status_code, message, attempt_count, circuit_state

---

## Failure Conditions

- Circuit breaker missing HALF_OPEN state
- No jitter in backoff (thundering herd problem)
- Raises raw exceptions instead of structured errors
- Blocking I/O inside async function
