## Question

```
You are given 3 document chunks with source metadata:

[Source: AI-0003-OpenWebUI-Compatibility.md, Section 4.2]
"The parameter top_k is not supported by the NVIDIA NIM API and will 
be silently ignored or cause a 400 error."

[Source: NVIDIA-NIM-Docs-2024.pdf, Page 12]
"Supported parameters: temperature, top_p, max_tokens, presence_penalty, 
frequency_penalty, stop, seed, stream."

[Source: AI-0001-Nemotron-Engineering-Spec.md, Section 3.1]
"Nemotron Ultra 550B achieves state-of-the-art performance on MATH-500 
and GPQA benchmarks."

Question: "Can I use top_k with NVIDIA NIM?"

Generate a response that:
1. Answers correctly
2. Cites the specific sources that support the answer
3. Does NOT cite irrelevant sources
4. Is concise
```

---

## Expected Behaviour

**Correct response:**
> No, `top_k` is not supported by the NVIDIA NIM API. According to the internal audit (AI-0003-OpenWebUI-Compatibility.md, Section 4.2), it will be silently ignored or cause a 400 error. This is consistent with NVIDIA's official documentation (NVIDIA-NIM-Docs-2024.pdf, Page 12), which lists the supported parameters and does not include `top_k`.

Key points:
- Cites exactly AI-0003 and NVIDIA docs — NOT AI-0001 (irrelevant to this question)
- Does not include the benchmarks information (AI-0001 content)
- Answer is concise
- Cites section/page for credibility

---

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Correct answer (top_k not supported) | 30% | Non-negotiable |
| Cites AI-0003 and NVIDIA docs | 35% | Specific source attribution |
| Does NOT cite AI-0001 | 20% | Irrelevant source exclusion |
| Concise response | 15% | No padding |

---

## Failure Conditions

- Answers "yes, top_k is supported"
- Cites AI-0001 as support for the answer
- Does not cite any source
- Long, padded response
