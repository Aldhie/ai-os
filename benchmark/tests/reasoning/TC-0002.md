## Question

```
A software system has these components:
- Service A calls Service B with a 200ms timeout
- Service B calls Database D with a 500ms timeout  
- Service B also calls Cache C (read-through, 50ms timeout)
- Database D has been experiencing 450ms average latency (P95: 800ms)
- Cache C hit rate is 30%

A user reports that their requests are timing out approximately 70% of the time.

Trace the complete causal chain that explains the 70% failure rate.
Identify which single change would have the highest impact on reducing failures.
Quantify the expected improvement from your recommended change.
```

---

## Expected Behaviour

1. Model identifies the causal chain: user request → A → B → {C miss + D latency}
2. Model calculates: 70% cache miss (100%-30%) × DB P95 latency (800ms) > B→D timeout (500ms) → B fails → A's 200ms timeout is irrelevant (B fails before it)
3. Model correctly identifies that increasing cache hit rate OR reducing DB latency are the primary levers
4. Model recommends improving cache hit rate as highest impact: 70% miss rate × failures → improving cache to 70% hit rate would reduce DB calls to 30% of current → failures drop from ~70% to roughly 70% × 0.30 = ~21%
5. Model quantifies: from 70% failure → approximately 21% failure = 50 percentage point improvement

## Evaluation Criteria

| Criterion | Weight | Description |
|-----------|--------|-------------|
| Correct causal chain | 35% | All links identified correctly |
| Correct failure rate attribution | 25% | Math reasoning is sound |
| Valid recommendation | 25% | Recommends cache or DB fix with justification |
| Quantified improvement | 15% | Provides numeric estimate |

---

## Scoring

| Score | Condition |
|-------|-----------|
| 10/10 | All criteria met with correct numerics |
| 8/10 | Correct chain and recommendation, minor numeric error |
| 6/10 | Correct recommendation, incomplete causal chain |
| 4/10 | Identifies cache as issue but no quantification |
| 2/10 | Partially correct analysis, wrong recommendation |
| 0/10 | Incorrect or no meaningful analysis |

---

## Failure Conditions

- Model recommends increasing Service A or B timeout as primary fix
- Model ignores cache miss rate as root cause
- No quantification of impact

---

## Success Conditions

- Correct identification of cache miss × DB latency interaction
- Recommendation targets root cause
- Improvement is quantified numerically
