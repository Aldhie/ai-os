## Question

```python
# Bug report: This service grows from 200MB to 4GB over 24 hours.
# Find ALL bugs and explain the root cause of the memory leak.

import asyncio
import aiohttp
from collections import defaultdict

class MetricsCollector:
    def __init__(self):
        self.request_logs = []  # stores every request ever made
        self.session = aiohttp.ClientSession()
        self.counters = defaultdict(int)

    async def record(self, endpoint: str, duration: float, response_body: bytes):
        self.request_logs.append({
            'endpoint': endpoint,
            'duration': duration,
            'body': response_body,  # full response stored
            'timestamp': asyncio.get_event_loop().time()
        })
        self.counters[endpoint] += 1

    async def get_stats(self) -> dict:
        return {
            'total': len(self.request_logs),
            'by_endpoint': dict(self.counters),
            'recent': self.request_logs[-100:]  # last 100 only
        }

class APIService:
    def __init__(self):
        self.collectors = {}  # per-user collector

    async def handle_request(self, user_id: str, endpoint: str):
        if user_id not in self.collectors:
            self.collectors[user_id] = MetricsCollector()
        
        collector = self.collectors[user_id]
        async with collector.session.get(endpoint) as resp:
            body = await resp.read()
            await collector.record(endpoint, 0.1, body)
        return body

service = APIService()
```

Identify every memory leak, explain the root cause, and provide corrected code.
```

## Expected Behaviour

Model identifies ALL of the following:

1. **`request_logs` grows unbounded** — stores every request body indefinitely. Fix: use `deque(maxlen=1000)` or store only metadata (not full body).
2. **`response_body: bytes` stored in full** — if responses are large (e.g., 10KB each), 1M requests = 10GB. Fix: store only summary metadata.
3. **`aiohttp.ClientSession` created per user, never closed** — each session holds a connection pool + SSL context. Fix: single shared session or explicit `__aenter__`/`__aexit__`.
4. **`self.collectors` grows unbounded** — one collector per user_id, never evicted. If 1M unique users → 1M collectors each with open sessions. Fix: LRU eviction or time-based cleanup.
5. **`asyncio.get_event_loop()` deprecated in Python 3.10+** — should use `asyncio.get_running_loop()`. (Minor but real.)
